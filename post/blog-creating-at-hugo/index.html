Hugoとは

HugoとはGo言語で作られた静的サイトジェネレータです。
Markdownで記事を書き、ビルドを実行すると、あらかじめ作られたテンプレートのデザインを適用したCSSとHTMLを吐き出してサイトやブログを作ることができます。
他にも同様のツールとしてRuby製のJekyllやMiddleman、Node.js製のMetalsmithなどがあります。
Hugoの特徴は導入が簡単で、ビルドが早い。他のツールを使ったことはないですが、ビルドが一瞬で終わるのは確かです。

今回、Hugoを選んだのはブログサービスを使わずにブログを作るということに興味があったことと、少し前からGo言語を勉強していたためです。
ちなみにGo言語とはGoogleにより開発されたプログラミング言語で、コンパイル型の静的型付き言語です。

Hugoのインストール

参考: 公式サイト

Mac

brewを使ってインストールを行います。

$ ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
$ brew update
$ brew install hugo

インストール場所とバージョンは以下で確認できます。
現在の最新バージョンは v0.16 です。

$ which hugo
$ hugo version

プロジェクトの作成とテーマのダウンロード

Hugo Themes Site
全てのテーマをダウンロードすることもできますが、量が多いので既存のテーマを選んで一つダウンロードします。

Hugo Theme: Hugo Zen
今回はこちらの Hugo Zen を使いました。
また、テーマは自分で自由に作ることも可能です。

$ hugo new site <site name>
$ https://github.com/rakuishi/hugo-zen.git themes/hugo-zen

設定

config.tomlの設定

テーマによって設定が異なるので(表示する情報が異なるため)、上記のテーマごとのページや /themes/<任意のテーマ>/README.md などを参考にして設定を行います。

baseurl = "http://38fanjia.github.com/"
languageCode = "ja"
title = "だれも聞いていないと思って歌え"
author = "38fanjia"
copyright = "38fanjia All rights reserved."

[params]
  logo      = "/images/icon.jpg"
  twitter   = "https://twitter.com/38fanjia"
  github    = "https://github.com/38fanjia/"

不要な設定はプロパティを空にしておいても問題ありません。
disqusShortname は記事へのコメント機能を追加するためのサービスです。google analyticsと合わせて今回は設定しません。

logo画像は、 content/images/ 下に配置しています。

確認

hugo server コマンドで、テーマを指定してプレビューします。

% hugo server --theme=hugo-zen --buildDrafts --watch

http://localhost:1313/
上記URLにアクセスすると、記事が何もないブログが表示されます。

プレビューは、変更を加えるたびに最新状態に表示を更新してくれます。
また、 control + c で止めることができます。

記事を作成する

% hugo new post/hello-world.md -f ./themes/hugo-zen/archetypes/default.md

-f でfrontmatterのテンプレートを指定して記事を作成します。
frontmatterとはTOMLやYAMLなどで作成されたフォーマットのことで、自動的に必要なプロパティを設定して記事ファイルを生成してくれます。

今回の場合は下記のように生成されるので、本文をこのテンプレートの下に記載する形になります。
また、記事タイトルは hugo new xxx の際に指定したファイル名が適用されるのでこちらも必要に応じて修正します。

+++
date = "2016-05-22T16:41:02+09:00"
draft = true
title = "hello world"

+++

本文作成後、プレビューを実行すると下記のように表示されます。

サイトのビルド

% hugo -t hugo-zen

テーマを指定してビルドを実行します。
publicディレクトリが作成され、公開用のファイルが生成されていることが確認できます。
これらのファイルは、Github Pagesなどにそのままアップして公開することが可能です。
