Hugoで作成したサイトをGitHub Pagesで公開する

October 28, 2016

GitHub Pages とは

GitHub Pagesは、GitHubが提供するホスティングサービスです。
公開するページは xxx.github.io というURLになります(独自ドメインに変えることもできます)。

GitHub Pagesでは企業、団体、個人などがサイトを公開しています。

  • Facebook Code
  • Twitter ♥ Open Source
  • Geek Women Japan 2016
  • yoshiko-pg.github.io

ちなみに、公開するためのファイルの種類には限界があり、HTML + CSS + JavaScript が使用できます。
公開方法もリポジトリを作成して通常のGitと同様の操作を行うだけです。
また、GitHub Pagesで管理するリポジトリはプライベートにできません。

リポジトリの作成

GitHub Pages のリポジトリの作成方法に違いはほとんどなく、
Repository name: xxx.github.io
と「github.io」を含んだリポジトリ名にするだけとなります。
この時作成したリポジトリ名がサイトURLとなります。

また、公開向けだけではなく管理用に別途リポジトリを作成します。
今回こちらはプライベート管理したい&GitHubの有料会員ではないためBitbucketで管理することにします。

リモートリポジトリを追加する

公開用ではないHugoプロジェクトを、 xxx.github.io とは異なるリポジトリで管理し、
公開用のファイル(後述します)を xxx.github.io のリポジトリで管理を行います。

% cd <hugo site>
% git remote add origin <repository clone url>
% git submodule add git@github.com:<user name>/<github pages repository>.git public

公開するファイルが public ディレクトリに生成されるため、github pagesで管理するディレクトリに public を指定します。

サイトのビルドをし、公開用のファイルを生成する

下記のようにテーマを指定してビルドを実行すると、新たにpublicフォルダが生成されます。
publicフォルダ内には、公開用のhtmlファイルなどが格納されています。

% hugo --theme=<theme name>
% tree -L 1 public
public
├── 404.html
├── css
├── images
├── index.html
├── index.xml
├── post
└── sitemap.xml

公開する

管理用リポジトリと公開用リポジトリは、コミットとリモートリポジトリへのpushを別々に実行する必要があります。
